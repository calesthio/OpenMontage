# 赤灵AI运营工作台 v5 设计说明

## 设计方向

v5 从 v4 的“赤红流光驾驶舱”收敛为更耐看的简约系统。参考苹果式产品设计原则：浅色背景、强留白、低装饰、清晰层级、柔和阴影、克制动效。目标是让运营人员长时间使用不会疲劳，同时保留必要的状态反馈。

## 为什么改

- v4 的深色、网格和流光更像展示大屏，短时间有冲击力，但长期使用容易疲劳。
- 运营工作台需要“安静、确定、可信赖”，不需要过多视觉兴奋。
- 反馈仍然重要，但应该通过按钮状态、Toast、进度环、任务抽屉来表达，而不是靠强特效。

## v5 页面

0. **登录页**
   - 企业内部入口，支持账号密码和验证码登录。
   - 右侧展示工作台概览，登录前也能知道进入后要处理什么。
   - 强调团队授权和素材安全，不展示任何内部模型或接口信息。

1. **生产台**
   - 快速导入、当前任务、最近作品。
   - 使用浅色大卡片和少量状态点表达任务状态。

2. **创建作品**
   - 三步流程清楚但不抢眼。
   - 参数项保持可编辑：15 秒内、标准 480p / 高清 720p、默认 1 条、单批最多 5 条、口播短句。
   - 右侧预览区用于素材裁切和字幕位置。

3. **生成中反馈**
   - 点击“确认并生成”后立即进入生成态。
   - 保留 Toast、按钮 loading、进度环、任务抽屉、预计时间。
   - 动效克制：轻微呼吸、旋转 loading，不做炫光。

4. **人工审核**
   - 左侧预览，中间文案，右侧审核侧栏。
   - 审核项明确：素材授权、肖像授权、字幕规则、画面方向。

5. **成品交付**
   - 下载成片、字幕、审核记录。
   - 提供交付包下载和分享路径复制。

## 落地建议

推荐以 v5 作为 Web 工作台主视觉方向。v4 可以作为营销页或发布演示大屏风格，不建议作为日常后台主界面。

前端实现建议：

- 使用 Web App 首发。
- 用系统字体和 CSS 变量，不引入复杂视觉库。
- 动效使用 CSS transition / keyframes 即可。
- 任务状态通过后端轮询或 WebSocket 更新。
- 目标机生产能力通过 Worker 接入，Web 只负责任务下发和状态展示。
