# Reference Video Analysis Package Design

## Goal

Build MVP1 for a creator workflow that turns a Douyin link or local video file into an editable replication package. The package helps a human review and modify the source video's copy, scene structure, pacing, and production strategy before any face replacement, digital-human generation, Seedance generation, or final editing starts.

This MVP is analysis-only. It does not directly replace faces, impersonate people, publish outputs, or auto-generate final videos.

## Product Boundary

The first version supports two input paths:

- **Douyin link first**: try to download or resolve the reference video from a user-provided link.
- **Local file fallback**: if link download fails because of login, platform protection, region, watermark handling, or network restrictions, ask the user to provide a local video file path.

The output is a human-editable "replication package" with:

- Source metadata and provenance.
- Raw transcript extracted from the video's audio.
- Editable rewrite draft derived from the transcript.
- Scene table with start/end times, visual descriptions, speech snippets, pacing notes, and production hints.
- Keyframe references for visual review.
- Recommended next production mode: direct face replacement, full remake, or hybrid remake.
- Safety and rights checklist confirming that downstream face assets must be team-authorized.

## User Flow

1. User provides a Douyin URL or local video path.
2. The system creates a project workspace under `projects/<slug>/`.
3. The system downloads or imports the reference video.
4. FFmpeg extracts audio, probes video metadata, and samples frames.
5. Existing analysis tools segment the video and identify scene boundaries.
6. Speech transcription produces a raw transcript.
7. A structured package is generated for human review.
8. The user edits copy, scene notes, and production mode.
9. Only after approval does the result move to later production pipelines.

## Architecture

### Pipeline

Add a new focused pipeline manifest named `reference-video-analysis`. Keep it separate from `creator-video` because it has one clear responsibility and can feed either the future direct-face-swap path or the existing creator-video remake path.

Stages:

1. **ingest**: resolve Douyin link or import local video.
2. **analyze**: extract audio, metadata, scene boundaries, and keyframes.
3. **transcribe**: generate raw transcript and timed speech segments.
4. **package**: build the replication package and strategy recommendation.
5. **review**: require human confirmation before downstream production.

### Tools

Reuse existing OpenMontage capabilities where possible:

- `video_downloader` for supported URL ingestion.
- `custom_asset_import` for local fallback video import.
- `scene_detect` for shot or scene boundaries.
- `frame_sampler` for keyframes.
- `transcriber` or available transcription tool for speech-to-text.
- `video_analyzer` for broader metadata and visual analysis where available.
- FFmpeg/ffprobe for local media probing, audio extraction, and frame operations.

Add one new orchestration tool only if existing tools cannot produce a stable combined artifact:

- `reference_video_package`: reads outputs from ingest/analyze/transcribe and writes one normalized JSON package.

## Replication Package Schema

The package is JSON plus a Markdown review file generated by the packaging tool. JSON is the machine handoff contract; Markdown is the human editing surface.

```json
{
  "version": "1.0",
  "source": {
    "input_type": "douyin_url",
    "input": "https://...",
    "local_video_path": "projects/example/reference/source.mp4",
    "duration_seconds": 38.2,
    "width": 1080,
    "height": 1920,
    "fps": 30
  },
  "transcript": {
    "raw_text": "原视频转写文案",
    "segments": [
      {
        "start": 0.0,
        "end": 2.8,
        "text": "开头钩子"
      }
    ]
  },
  "rewrite_draft": {
    "status": "needs_human_edit",
    "text": "可人工修改的复刻稿"
  },
  "scenes": [
    {
      "scene_id": "s1",
      "start": 0.0,
      "end": 3.2,
      "visual_summary": "人物正面口播，办公室背景",
      "speech": "开头钩子",
      "camera_motion": "固定机位，轻微推近",
      "pacing": "fast_hook",
      "keyframes": ["projects/example/keyframes/s1_001.jpg"],
      "production_hint": "team_digital_human_remake"
    }
  ],
  "replication_strategy": {
    "recommended_mode": "hybrid",
    "alternatives": ["direct_face_swap", "full_remake"],
    "reason": "口播适合团队数字人重制，B-roll 可用 Seedance 重制"
  },
  "approval": {
    "status": "pending_human_review",
    "required_before_production": true
  }
}
```

## Compliance and Safety Rules

- Treat the input video as a reference artifact, not a license to republish a copy.
- Do not start face replacement or digital-human generation inside MVP1.
- Downstream production must use only team-authorized face or avatar assets.
- Preserve provenance: source URL or local file, transcript origin, keyframes, and generated package path.
- Mark rewrite output as a draft requiring human review.
- If URL download fails, surface a structured fallback message instead of scraping around platform protections.

## Error Handling

- Invalid URL: return a clear validation error and ask for a local file.
- Download blocked: create a partial package with failure reason and local-file instructions.
- No audio: still generate visual scene analysis and mark transcript unavailable.
- Transcription unavailable: create the scene/keyframe package and mark transcript pending.
- Scene detection failure: fall back to fixed-interval segmentation.
- Missing FFmpeg: block with installation instructions.

## Testing Plan

- Contract test that the pipeline manifest loads and lists all required stage skills.
- Unit test that `reference_video_package` normalizes analysis outputs into the schema.
- Fixture test with a short local MP4 that has audio and two visual sections.
- Fallback test for a local MP4 with no audio.
- URL failure test that returns a local-file fallback instead of crashing.
- Registry test confirming all required tools are discoverable or optional with clear blockers.

## Non-Goals

- No direct face replacement implementation.
- No digital-human API integration.
- No automatic publishing.
- No bypassing Douyin login, anti-scraping, watermark, or access restrictions.
- No guarantee that a source video's exact wording, likeness, music, or edit is legally reusable.

## Implementation Decisions

- Pipeline name: `reference-video-analysis`.
- Review artifact: `reference_video_package` generates both JSON and Markdown.
- Transcription: use the existing `transcriber` tool when available; if unavailable, package visual analysis and mark transcript as `pending_transcription`.
