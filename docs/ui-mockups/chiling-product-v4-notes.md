# 赤灵AI运营工作台 v4 设计说明

## v4 方向

v4 从“普通管理后台”升级为“内容生产驾驶舱”。视觉关键词是赤红流光、深色玻璃、任务抽屉、实时反馈。用户不需要知道内部模型、接口、供应商或渲染技术，只需要知道现在做到了哪一步、是否需要自己确认、什么时候能下载。

## 页面

1. **生产驾驶舱**
   - 展示快速导入、当前生产任务、最近作品和任务进度。
   - 适合运营人员每天打开后直接进入工作。

2. **创建作品**
   - 三步流程：导入参考、设置生成、提交审核。
   - 保留关键参数：15 秒内、默认 1 条、单批最多 5 条、标准 480p / 高清 720p。
   - 右侧实时预览素材裁切和字幕位置。

3. **生成中反馈**
   - 点击“确认并生成”后，必须立即反馈。
   - 反馈组合：按钮 loading、顶部 Toast、进度环、任务抽屉、实时预览、预计剩余时间。

4. **人工审核**
   - 主页面保留视频预览和文案编辑。
   - 右侧审核详情侧栏用于确认授权、字幕规则、画面方向。

5. **成品交付**
   - 下载成片、字幕文件、审核记录。
   - 下载交付包前提示再次确认授权和平台规范。

## 生成反馈规则

- 用户点击提交后，按钮进入 loading，不允许重复点击。
- 立即显示 Toast：任务已进入队列。
- 右侧抽屉从“审核项”切换成“任务阶段”。
- 阶段包括：解析参考、整理文案、生成画面、合成字幕、质检交付。
- 每个阶段展示状态：已完成、进行中、等待、失败重试。
- 生成失败时不静默：显示失败原因、自动重试状态和人工重试按钮。

## 推荐落地形态

推荐先做 **Web 工作台**，不是一开始做纯客户端。

原因：

- 团队审核、素材库、项目记录、交付包下载更适合 Web。
- 多人协作和权限管理放在 Web 端更稳定。
- 生成任务可以由后端队列和 Worker 处理，用户页面只负责展示状态。
- 如果目标机需要本地跑任务，可以安装本地 Worker，由 Web 端下发任务。
- 后续需要桌面版时，再用 Tauri/Electron 包一层 Web 前端即可。

建议架构：

```text
Web 前端
  -> 后端 API / 任务队列
  -> 生产 Worker（云端或目标机）
  -> 文件存储 / 交付包
```
